# 三国立志传：草莽英雄 v2.0 改进设计

## 改进目标
- 架构：事件抽离为外部JSON配置
- 数值：放慢游戏成长节奏
- UI：优化信息展示，解决拥挤问题
- 补充：开放出身选择，增加更多事件

## 一、架构改进：外部事件JSON

### 变更
1. 创建 `events.json` 文件，将 `game.js` 中 `game.events` 数组迁出
2. `game.init()` 增加 `fetch('./events.json')` 异步加载
3. 保持事件数据格式不变，兼容现有逻辑

### 事件格式
```json
[
  {
    "event_id": "evt_001",
    "trigger_type": "random",
    "title": "黄巾残党过境",
    "description": "一伙大约二十人的黄巾残党路过你的领地，要求你交纳保护费。",
    "opt1_text": "【武力硬刚】 武力+士兵 > 80",
    "opt1_attr": "force",
    "opt1_diff": 80,
    "opt1_success_text": "你击败了黄巾残党，缴获 150 钱",
    "opt1_success_reward": {"money": 150, "exp_force": 5},
    "opt1_fail_text": "你击退了黄巾，但被抢走一半粮食",
    "opt1_fail_reward": {"grain": "Math.floor(game.res.grain / 2)"},
    "opt2_text": "【破财消灾】 交出 100 粮食，安全过关",
    "opt2_success_text": "黄巾拿了粮食走了，你安全过关",
    "opt2_success_reward": {"grain": -100}
  }
]
```

### 注意
- 奖励支持JSON表达式，需要在 `applyReward` 中保持eval逻辑
- 加载失败降级到内置默认事件

## 二、数值调整：放慢节奏

| 项目 | 当前 | 调整后 |
|------|------|--------|
| 基础行动力 | 5 | 4 |
| 单次行动经验 | 8-15 | 5-10 |
| 升级所需经验 | 100 × lv | 120 × lv |
| 升级属性加成 | +3 | +2 |
| 随机事件概率 | 30% | 25% |

## 三、UI信息展示优化

### 顶部状态栏重构
- 原来两行：第一行日期+AP，第二行属性+资源共8个 → **改为三行**
  1. 日期 + 行动力
  2. 四大属性（武力/智力/魅力/统率）
  3. 四大资源（金钱/粮食/人口/士兵）
- 增加行间距，呼吸感更好

### 其他UI调整
- 地点卡片增加内边距，缓解拥挤
- 角色面板每个属性都显示经验条
- 增加领地信息卡片在角色面板
- 日志滚动到底部自动

## 四、玩法补充

### 开放出身选择
- 新游戏开局弹出选择框
- 三个选项：流民/退役兵士/落第书生（已有数据）
- 根据选择分配初始属性和资源

### 新增事件
- 在现有2个基础上，新增18个事件，总共20个
- 覆盖各种场景：贼寇、商人、名士、天灾、奇遇等
